# examples

## Purpose

Reserved **containers** for validated, structured outputs of the framework: pioneer studies, successful and failed project studies, case studies, and benchmarks.

## Why This Folder Exists

The framework must be testable against reality. These folders are the slots where completed, validated knowledge artifacts will live — proving the framework works end-to-end.

## Source of Truth

Populated later from validated research; empty containers for now.

## Input

Validated knowledge artifacts produced via the templates and pipeline.

## Output

Reference examples and benchmarks for the framework.

## Consumer

Framework users, reviewers, and reasoning-validation processes.

## Folder Structure

```
examples/
├── DatasetIndex.md   # Curation record: distribution, gaps, candidate queue, bibliography
├── Pioneer/          # Category-defining projects (Batch 01 populated)
├── Successful/       # Validated success studies
├── Failed/           # Validated failure studies
├── CaseStudies/      # In-depth structured case studies (Batch 01 populated)
└── Benchmarks/       # Reference benchmarks
```

Curated knowledge currently loaded: **Batch 01** (8 pioneer projects, source: Deep Research / Gemini)
and **Batch 02** (10 pioneer projects, source: web research) — **18 projects total**. See
`DatasetIndex.md` for the full record, cumulative distribution, gap status, and bibliography.

## Workflow Position

Represents the `Applications` end of `Research → Knowledge → Ontology → Patterns → Reasoning → Framework → Applications` — where the framework is exercised.

## Rules

1. Containers only until validated artifacts exist.
2. Any content must follow the templates and schema.
3. No unvalidated or speculative data.

## Naming Convention

Subfolders in `PascalCase`; artifacts follow the relevant template naming.

## Future Expansion

Curated libraries of studies and quantitative benchmark suites.
