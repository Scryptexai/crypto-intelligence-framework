# archive

## Purpose

Holds **deprecated or superseded material** — documentation, structures, or drafts that are no longer active but retained for traceability.

## Why This Folder Exists

A knowledge repository needs memory of what changed. Rather than deleting outdated structures, they are moved here so history and rationale remain auditable.

## Source of Truth

Superseded versions of documents/structures moved from active folders.

## Input

Deprecated files relocated from elsewhere in the repository.

## Output

An auditable record of superseded material.

## Consumer

Maintainers and auditors reviewing the evolution of the framework.

## Folder Structure

```
archive/
└── (dated/labelled deprecated material added later)
```

## Workflow Position

Outside the active pipeline — preserves prior states of any stage.

## Rules

1. Only superseded/deprecated material.
2. Never referenced by active reasoning.
3. Preserve original context when archiving.

## Naming Convention

Prefix with a date or version label, e.g. `YYYY-MM-DD-Name.md`.

## Future Expansion

A structured, indexed archive with deprecation notes.
