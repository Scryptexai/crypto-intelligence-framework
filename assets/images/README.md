# images

## Purpose

Container for supporting images for documentation.

## Why This Folder Exists

General images referenced by documentation files. Empty until assets are added.

## Source of Truth

Authored/exported by maintainers; not knowledge.

## Input

Source or exported media files.

## Output

Referenceable assets for documentation.

## Consumer

Documentation readers.

## Folder Structure

```
images/
└── (asset files added later)
```

## Workflow Position

Cross-cutting documentation support.

## Rules

1. Support docs only.
2. Descriptive filenames.
3. No data/knowledge in assets.

## Naming Convention

Lowercase, hyphenated filenames.

## Future Expansion

A maintained set of supporting images for documentation.
