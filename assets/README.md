# assets

## Purpose

Holds **supporting visual assets** for the documentation: diagrams, images, and icons.

## Why This Folder Exists

Documentation is clearer with visuals. This folder centralises non-textual assets so they can be referenced consistently across the repository.

## Source of Truth

Created by maintainers to support documentation; not knowledge.

## Input

Design/diagram sources and exported media.

## Output

Referenceable visual assets embedded in documentation.

## Consumer

Readers of the documentation and README files.

## Folder Structure

```
assets/
├── diagrams/   # Architecture and pipeline diagrams
├── images/     # Supporting images
└── icons/      # Icons and small graphics
```

## Workflow Position

Cross-cutting support — illustrates every stage of the pipeline.

## Rules

1. Assets support docs only.
2. No data or knowledge encoded as assets.
3. Keep filenames descriptive.

## Naming Convention

Lowercase, hyphenated descriptive filenames.

## Future Expansion

Interactive diagrams and a shared visual style guide.
