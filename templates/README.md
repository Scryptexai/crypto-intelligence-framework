# templates

## Purpose

Holds **reusable capture templates** that standardise how research, projects, ontology entries, patterns, frameworks, and case studies are recorded.

## Why This Folder Exists

Consistent knowledge requires consistent capture. Templates enforce the documented structure at the point of data entry, feeding clean input into the pipeline.

## Source of Truth

Authored by maintainers to standardise capture; not derived from research.

## Input

The schema and ontology specifications.

## Output

Blank, structured templates ready to be filled by research.

## Consumer

Researchers and extraction agents.

## Folder Structure

```
templates/
├── ResearchTemplate.md
├── ProjectTemplate.md
├── OntologyTemplate.md
├── PatternTemplate.md
├── FrameworkTemplate.md
└── CaseStudyTemplate.md
```

## Workflow Position

Feeds the front of `Research → Knowledge → Ontology → Patterns → Reasoning → Framework → Applications` by shaping how knowledge enters.

## Rules

1. Structure and placeholders only.
2. Must align with Schema/Ontology.
3. No real data.

## Naming Convention

`*Template.md` in `PascalCase`.

## Future Expansion

Additional templates for new knowledge types.
