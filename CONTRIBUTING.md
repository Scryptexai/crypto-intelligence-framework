# CONTRIBUTING

## Purpose

Document *how to contribute* to the repository — the rules and workflow for adding structure and, later, knowledge, while preserving the self-documenting design.

## Description

This file is a documentation container only. It defines *how* knowledge must be captured — it must not contain knowledge, real project data, examples, or history. Actual content is produced downstream by Deep Research and inserted under this specification. It specifies contribution rules; it is not a knowledge file. It tells contributors how to keep the repository consistent.

## Why This File Exists

Contributions must not break the documented structure or mix knowledge into documentation. This file encodes those guardrails.

## Data Source

Maintainer conventions and the project's principles.

## Required Content

The following must be documented (format only):

- Branching and commit conventions
- Folder/file format requirements (README & file formats)
- The rule separating documentation from knowledge
- Review and validation expectations
- How to reference schema/templates when adding content

## Data Structure

Prose with clearly headed sections and checklists.

## Validation Rules

- Must reinforce the documentation-vs-knowledge boundary.
- Must reference the required README and file formats.
- Must stay consistent with `docs/Project/Principles.md`.
- No knowledge content.

## Used By

Contributors and AI agents modifying the repository.

## Related Files

`docs/Project/Principles.md`, `docs/Project/Scope.md`, root `README.md`.

## Future Expansion

Automated contribution checks and templates for pull requests.
