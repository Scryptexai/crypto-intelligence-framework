# Success

## Purpose

Documents the **success-factor dimensions** — VC backing, founder quality, community, developer activity, ecosystem, adoption, revenue, token, protocol, and airdrop factors — used to reason about why projects succeed.

## Why This Folder Exists

Success is multi-causal. This folder isolates each contributing factor so the reasoning layer can weigh them explicitly.

## Source of Truth

Ontology attributes plus outcome research.

## Input

Ontology attributes and outcome research.

## Output

Structured definitions of each success factor and its signals.

## Consumer

Reasoning, Valuation, and ExitStrategy layers.

## Folder Structure

```
Success/
├── VC.md          ├── Ecosystem.md   ├── Token.md
├── Founder.md     ├── Adoption.md    ├── Protocol.md
├── Community.md   ├── Revenue.md     └── Airdrop.md
└── Developer.md
```

## Workflow Position

Success sub-layer of the `Patterns` stage feeding `Reasoning` in `Research → Knowledge → Ontology → Patterns → Reasoning → Framework → Applications`.

## Rules

1. Define factor + signals only.
2. Signals must be observable.
3. No project outcomes stored.

## Naming Convention

`PascalCase.md`.

## Future Expansion

Quantified signal inputs and factor-interaction modelling.
