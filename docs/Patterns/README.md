# Patterns

## Purpose

Documents the **recurring behavioural and economic patterns** in crypto — flywheels, network effects, referral loops, liquidity dynamics, points, mining, ponzinomics, distribution, narrative, growth, failure, and recovery.

## Why This Folder Exists

Patterns are the reusable building blocks of reasoning-by-analogy. This folder defines each pattern so the engine can recognise and apply it across projects.

## Source of Truth

Candidate patterns from Extraction plus cross-project research.

## Input

Candidate patterns from extraction and research.

## Output

Generic, reusable pattern definitions.

## Consumer

Reasoning and MarketBehaviour layers.

## Folder Structure

```
Patterns/
├── Flywheel.md       ├── Mining.md        ├── Narrative.md
├── NetworkEffect.md  ├── Ponzinomics.md   ├── Growth.md
├── Referral.md       ├── Distribution.md  ├── Failure.md
├── Liquidity.md      └── Points.md        └── Recovery.md
```

## Workflow Position

Core of the `Patterns` stage in `Research → Knowledge → Ontology → Patterns → Reasoning → Framework → Applications`.

## Rules

1. Define patterns generically.
2. Explicit triggers and signals.
3. No project instances.

## Naming Convention

`PascalCase.md`, one pattern per file.

## Future Expansion

New patterns and automated pattern detection.
