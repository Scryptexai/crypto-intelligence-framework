# Mission

## Purpose

Specify how knowledge about **the concrete mission that pursues the vision** must be captured, structured, and validated within CIF's Project meta layer (Meta / Foundational).

## Description

This file is a documentation container only. It defines *how* knowledge must be captured — it must not contain knowledge, real project data, examples, or history. Actual content is produced downstream by Deep Research and inserted under this specification. It scopes exactly what belongs to the concrete mission that pursues the vision and nothing else, so the knowledge model stays modular and machine-readable.

## Why This File Exists

Within the pipeline (Research → Knowledge → Ontology → Patterns → Reasoning → Framework → Applications), the concrete mission that pursues the vision is a distinct unit that must be captured explicitly so downstream stages can reason over it consistently and traceably.

## Data Source

Authored by the CIF maintainers. Not derived from research.

## Required Content

The following must eventually be filled by research (documentation of fields only, not values):

- What CIF does day-to-day
- Who it serves
- The value it delivers
- Boundaries of the mission

## Data Structure

Prose documentation with clearly headed sections.

## Validation Rules

- Must describe intent, not knowledge.
- Must stay consistent with the documented pipeline.
- Must not contain project/case data.
- Must be human-readable and unambiguous.

## Used By

All contributors and AI agents that need to understand CIF's intent and boundaries.
